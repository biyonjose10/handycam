## Project Overview

Real-time webcam visual effect for **TouchDesigner 2023 (Python 3.11)**. MediaPipe
tracks both hands; their fingertips frame **three overlapping quadrilaterals**,
each rendering the webcam through a different print effect. Outside every quad is a
clean webcam passthrough with a hard edge. Each quad **disappears entirely** when
its hands aren't detected.

- **Quad A** â€” thumb + index tips â†’ **Risograph** (`riso_cmyk.frag`)
- **Quad B** â€” index + middle tips â†’ **Negative** (`negative.frag`)
- **Quad C** â€” middle + ring tips â†’ **Stippling** (`stipple_red.frag`)

Hand tracking uses the **`torinmb/mediapipe-touchdesigner`** GPU plugin, which
exposes a hands JSON DAT. The TD network is generated by a Python builder (a
`.toe` is binary and can't be authored as text).

## Key conventions / gotchas

- Shaders use TD GLSL-TOP convention: `out vec4 fragColor;`, `sTD2DInputs[]`, `vUV.st`, `uTD2DInfos[].res` (`.zw`=w,h `.xy`=1/w,1/h).
- Custom uniforms set via GLSL TOP **Vectors** page params: `vecNname`, `vecNvaluex/y/z/w` (0-indexed). In `quad_composite`: uC0â€“3 = slots 0â€“3, uGrain = 4, uActive = 5.
- Each quad's 4 corners = both hands' two fingertips, ordered CCW around their centroid (`_ordered` in `hands_to_chop.py`) for a clean convex polygon.
- **Presence gating:** a quad is only shown when both hands supply its 4 corners. `hands_to_chop.py` emits `presentA/B/C`; `quad_composite.frag` gates its inside branch on the `uActive` uniform. `uGrain.z` is a now-dead invert flag and the fallback slot for `uActive` if a TD build lacks vec5.
- Coordinate space: MediaPipe is top-left origin / y-down; TD UV is bottom-left / y-up. `FLIP_Y` and `MIRROR_X` in `config.py` reconcile this, baked into the GLSL uniform expressions (not a Math CHOP). `MIRROR_X` does NOT reverse quad winding â€” untested edge.
- `build_network.py` wraps parameter sets in a warn-on-missing helper so a single version-specific param name won't abort the whole build.
